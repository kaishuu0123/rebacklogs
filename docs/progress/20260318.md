### Kanban レイアウトバグ修正

#### 問題

Kanban 画面でタスクがすべて「新規」（最初の列）に表示される不具合。

#### 原因

`KanbanInner` コンポーネントの `<table>` 内で `<tbody>` が二重にネストされていた。

```tsx
// 修正前（不正な HTML）
<tbody>                          ← KanbanInner の tbody
  {filteredStories.map((story) => (
    <StoryRows ... />            ← StoryRows 自身も <tbody> を返す
  ))}
</tbody>

// 修正後
{filteredStories.map((story) => (
  <StoryRows ... />              ← <tbody> は StoryRows 内のみ
))}
```

`<tbody>` のネストは HTML 仕様上無効で、ブラウザがパースに失敗し全セルが 1 列目に集約されていた。Vue 移植時の構造ミス。

**変更ファイル**: `app/javascript/components/react/Kanban/index.tsx`

### i18n 対応（i18next + react-i18next）

Vue 時代は `vue-i18n` + `globals.json` を使用していたが、React への移植時に対応が抜けており、UI 文字列がすべて英語ハードコードになっていた。

#### 方針

- **Rails のロケール（`I18n.locale`）を `<html lang>` 経由で React に伝える**
- `i18next-browser-languagedetector` の `htmlTag` 検出を使い、ブラウザ言語設定ではなく Rails 側のロケール設定に追従させる
- `?locale=ja` 等の URL パラメータ変更も自動的に反映される

#### 新規ファイル

**`app/javascript/lib/i18n.ts`**
```ts
import i18n from 'i18next';
import LanguageDetector from 'i18next-browser-languagedetector';
import { initReactI18next } from 'react-i18next';
import en from '~/locales/en.json';
import ja from '~/locales/ja.json';

i18n
  .use(LanguageDetector)
  .use(initReactI18next)
  .init({
    resources: {
      en: { translation: en },
      ja: { translation: ja },
    },
    fallbackLng: 'en',
    detection: { order: ['htmlTag'] },
    interpolation: { escapeValue: false },
  });
```

**`app/javascript/locales/en.json`** — 英語翻訳（`globals.json` から移植 + React 追加分）

**`app/javascript/locales/ja.json`** — 日本語翻訳（同上）
