### 2. TicketModal タイトル truncate 修正

非常に長いタイトルでダイアログが横スクロールする問題を修正。

**原因:** `DialogContent` が CSS `grid` を使用しており、グリッドアイテムの `min-width` がデフォルト `auto` になっていた。

**修正:** タイトルを含む `flex gap-4` の div（グリッドの直接子）に `min-w-0` を追加。タイトル `h2` に `truncate min-w-0`。`overflow-hidden` は focus ring をクリップするため除去済み（後述）。

### 7. Kanban ドラッグ&ドロップ全面改善

#### ゴースト & ドラッグプレビュー
- **ゴースト:** `isDragging` 時に元の位置をダッシュ枠・薄背景のプレースホルダーとして表示（高さを `invisible` で維持）
- **ドラッグプレビュー:** `DragOverlay` に TaskCard と同レイアウトのカードを表示（`dropAnimation={null}` でドロップ後の戻りアニメなし）

#### クロスコンテナ drag の自然な gap インジケーター

**従来の問題:** ストーリーまたぎ・ステータスまたぎのドロップは `useDroppable` の "ドロップゾーン" 方式だったため、dnd-kit の自然な並び替えインジケーターが出なかった。

**解決策:** `localStories` のローカル状態を導入し `onDragOver` で optimistic 更新する方式に変更。

```
DndContext.onDragOver
  → setLocalStories でタスクを対象コンテナに移動
  → 対象セルの SortableContext にタスクが現れ、自然な gap インジケーターが動く
  → handleDragEnd で localStories の最終状態から index を読んで API 送信
```

**同一コンテナ内の並び替え:** `arrayMove` を使って `localStories` を更新（SortableContext の CSS transform 任せにせず、実配列を更新する）。

#### ドロップ後の一瞬フラッシュ修正

**原因:** `handleDragEnd` → `setActiveTaskId(null)` → `useEffect` が `localStories` を古いサーバーデータにリセット → API レスポンス前にフラッシュ。

**修正:** `useEffect` の sync 条件に `!updateTaskMutation.isPending` を追加。mutation 完了後のみ `localStories` をサーバーデータで更新する。

```ts
useEffect(() => {
  if (activeTaskId === null && !updateTaskMutation.isPending)
    setLocalStories(stories);
}, [stories, activeTaskId, updateTaskMutation.isPending]);
```

#### ドラッグハンドル & チケット番号クリック

- `{...listeners}` をカードルート div に移動 → カード全体がドラッグ面（`cursor-grab`）
- グリップアイコン（`GripVertical`）をチケット番号の左に薄く表示（`text-muted-foreground/30`、ハンドラなし）
- チケット番号を `<button>` にして `onClick` でダイアログを開く
- `onPointerDown={(e) => e.stopPropagation()}` でチケット番号エリアのドラッグ開始をブロック
